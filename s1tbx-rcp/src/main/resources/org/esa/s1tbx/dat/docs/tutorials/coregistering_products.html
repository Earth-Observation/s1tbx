Coregistration Tutorial

Coregistering Products

Image co-registration is fundamental for Interferometry SAR (InSAR) imaging and its applications, such as DEM map generation and analysis. To obtain a high quality InSAR image, the individual complex images need to be co-registered to sub-pixel accuracy.


The toolbox will accurately co-register one or more secondary images with respect to a reference image. The co-registration function is fully automatic, in the sense that it does not require the user to manually select ground control points (GCPs) for the reference and secondary images.


Images may be fully or only partly overlapping and may be from acquisitions taken at different times using multiple sensors or from multiple passages of the same satellite.


The achievable co-registration accuracy for images in the same acquisition geometry and over flat areas will be better than 0.2 pixels for two real images and better than 0.05 pixels for two complex images.


The image co-registration is accomplished in three major processing steps (see Figure 1) with three operators: Create Stack operator, GCP Selection operator and Warp operator.



Figure 1. Image co-registration

Input Images

The input images for the co-registration function can be complex or real. But all images must belong to the same type (i.e. they must all be complex or all real) and have the same projection system (all slant range or all ground range projected or all geocoded).   If the images are not in the same projection, the secondary image(s) should be reprojected into the same projection system as that of the reference image.

Create Stack

The Create Stack operator collocates the reference and secondary images. Basically the secondary image data is resampled into the geographical raster of the reference image. By doing so the reference and secondary images share the same geopositioning information and have the same dimension. For details of the Create Stack operator, readers are referred to Create Stack operator. For coregistration of detected products it is ok to use the resamling methods. For coregistration of complex products for interferometry, the option to not do any resampling in the CreateStack should be used.

GCP Selection

The GCP Selection operator then creates an alignment between reference and secondary images by matching the user selected reference GCPs to their corresponding secondary GCPs. There are two stages for the operation: coarse registration and fine registration. For real images co-registration, the coarse registration is applied. The registration is achieved by maximizing the cross-correlation between reference and secondary images on a series of imagettes defined across the images. For complex image co-registration, the additional fine registration is applied. The registration is achieved by maximizing the coherence between reference and secondary images at a series of imagettes defined across the images. For details of the GCP Selection operator, readers are referred to GCP Selection operator.

Warp

With the reference-secondary GCP pairs selected, a warp function is created by the Warp operator, which maps pixels in the secondary image into pixels in the reference image. For details of the Warp operator, readers are referred to Warp operator.

Co-Registration from the DAT

   Detailed steps of using this operator are given below:
  1. From the SAR Tools menu, select Coregistration.
  2. Add a reference product to the ProductSet Reader.
  3. Add any secondary products to the ProductSet reader or drag and drop products from a Project.
  4. In the CreateStack tab, select one source band from the reference and one source band from the secondary for detected products. If you are using complex data products, select bands i and q for the reference and select bands i and q from the secondaries.  For interferometric processing, select no resampling and Reference extents in the CreateStack.
  5. In the GCP Selection tab enter the number of GCPs to create.
  6. In the Warp tab select the RMS threshold and warp polynomial order.
  7. In the Target tab enter in the output product name and location.
  8. Press the Run button to begin processing.
